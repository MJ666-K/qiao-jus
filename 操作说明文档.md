# 枫桥智诉 · 操作说明文档

本文档面向**最终用户**与**运维人员**，说明如何部署、登录、使用各功能模块及常见问题处理。

---

## 一、系统访问

### 1.1 访问地址

| 环境 | 地址 | 说明 |
|------|------|------|
| 生产部署 | `http://<服务器IP>/` | 仅开放 80 端口，经 Nginx 统一入口 |
| 本地开发 | `http://localhost:5173` | Vite 开发服务器 |
| API 文档（生产） | `http://<服务器IP>/api/docs` | Swagger，需网络可达 |
| API 文档（本地） | `http://localhost:8000/docs` | 直连后端 |

### 1.2 账号与登录

- 登录页需填写：**邮箱、密码、图形验证码**
- 密码规则：**至少 8 位**（注册时校验）
- 测试账号（需先在服务器创建）：
  - 邮箱：`seed@demo.com`
  - 密码：`seed12345`

**生产环境首次使用**（数据库为空，无预置账号）：

```bash
cd deploy
./scripts/bootstrap_user.sh
# 或自定义: ./scripts/bootstrap_user.sh 你的邮箱@example.com 你的密码8位 admin
```

也可在登录页点击「立即注册」自助注册（默认普通用户）。

### 1.3 角色说明

| 角色 | 权限 |
|------|------|
| **普通用户** | 文档上传、报告生成、智能助手、我的报告 |
| **管理员** | 上述全部 + 检索测试、知识图谱、用户管理、系统配置 |

管理员可在 **系统配置 → 用户管理** 中提升/降级用户角色。

## 二、功能模块操作指南

### 2.1 概览

登录后默认进入概览页，展示文档数量、报告统计、最近活动等摘要信息。

### 2.2 我的文档

**用途**：上传、管理个人法律文档（合同、材料等），供检索与问答使用。

**操作步骤**：

1. 进入 **我的文档**
2. 选择或新建**知识库**（数据集）
3. 拖拽或点击上传 `.md` / `.pdf` / `.docx` 文件
4. 等待文档状态变为 **done**（后台 Worker 自动解析、切块、向量化、建图谱）
5. 可选：点击文档进入 **文本块管理**，查看/编辑切块内容

**注意**：

- 上传依赖阿里云 OSS 配置，未配置时上传会失败
- 处理中的文档状态为 `processing`，失败为 `failed`，可重新上传

### 2.3 生成报告

**用途**：基于已上传文档，AI 自动生成结构化法律分析报告（风险点、法条引用、建议等）。

**操作步骤**：

1. 进入 **生成报告**
2. 选择报告类型（如合同审查、劳动合规等）
3. 选择关联的知识库与文档
4. 提交生成，等待完成
5. 在 **我的报告** 中查看详情，可继续追问或导出

### 2.4 我的报告

查看历史报告列表，点击进入报告详情页，支持：

- 阅读结构化分析内容
- 查看引用来源（法条、类案、用户材料等）
- 跳转到智能助手继续追问

### 2.5 我的助手（智能问答）

**用途**：创建专属 AI 助手，绑定知识库与报告，进行多轮法律问答。

**界面布局**（三栏）：

| 栏位 | 功能 |
|------|------|
| 助手列表 | 创建/选择助手，绑定知识库与报告 |
| 会话列表 | 同一助手下多个独立对话 |
| 对话区 | 流式问答、引用来源、推荐追问 |

**操作步骤**：

1. 进入 **我的助手**
2. 点击 **新建助手**，填写名称，可选绑定知识库、报告
3. 选中助手后 **新建会话**
4. 在输入框提问，**Ctrl + Enter** 发送
5. 查看回答中的引用来源，点击推荐问题继续追问

**说明**：

- 未绑定知识库时，助手直接与大模型对话（无文档召回）
- 绑定报告后，问答可结合报告上下文

### 2.6 检索测试（管理员）

**用途**：调试混合检索效果，查看 Dense + BM25 + RRF + 重排 的召回结果。

输入查询语句，选择知识库，查看返回的文本块及分数。

### 2.7 知识图谱（管理员）

**用途**：可视化实体关系，支持按知识库加载全库关系图。

**操作**：

1. 进入 **知识图谱**
2. 选择知识库，自动加载关系图
3. 点击实体节点查看详情
4. 可执行局部/全局图谱检索

图谱数据来自文档入库时的 LLM 实体关系抽取（Neo4j 存储）。

### 2.8 系统配置（管理员）

| 标签页 | 功能 |
|--------|------|
| 当前账号 | 查看自己的邮箱、角色、权限 |
| 用户管理 | 列出用户、提升/降级管理员 |
| 检索与模型 | 切块参数、检索参数、重排模型、LLM 温度等 |

**配置生效说明**：

- 检索、重排、模型温度：保存后**立即生效**（进程内更新）
- 切块参数：仅对**新上传文档**生效，存量需重新索引

## 三、部署与运维

### 3.1 一键部署

```bash
# 1. 配置环境
cp deploy/.env.example deploy/.env
# 编辑：LLM_API_KEY、JWT_SECRET_KEY、OSS 等

# 2. 部署
cd deploy
chmod +x scripts/*.sh
./scripts/deploy.sh

# 3. 创建首个管理员
./scripts/bootstrap_user.sh
```

### 3.2 部署脚本选项

```bash
./scripts/deploy.sh              # 构建前端 + 后端 + 启动（默认）
./scripts/deploy.sh --up-only    # 仅启动容器，跳过构建
./scripts/deploy.sh --rebuild-backend  # 强制重建后端镜像
./scripts/deploy.sh --help       # 查看帮助
```

### 3.3 目录结构

```
deploy/
├── .env              # 生产环境变量（Docker 内网服务名）
├── app/              # 前端静态文件（Nginx 根目录）
├── data/             # 持久化数据
│   ├── postgres/
│   ├── qdrant/
│   ├── neo4j/
│   └── redis/
├── nginx/            # Nginx 配置
└── scripts/          # 部署脚本
```

### 3.4 常用运维命令

```bash
cd deploy

docker compose ps                    # 查看容器状态
docker compose logs -f api           # API 日志
docker compose logs -f worker        # 异步任务日志
docker compose logs -f nginx         # 前端/Nginx 日志
docker compose restart api worker    # 重启后端
docker compose down                  # 停止（数据保留）
```

### 3.5 更新部署

```bash
# 只更新了前端
./scripts/deploy.sh                  # 会重新 npm build 并复制到 app/

# 只改了 Python 业务代码
./scripts/deploy.sh                  # Docker 层缓存，pip 依赖不会重装
docker compose up -d api worker      # 重启 api/worker 加载新镜像

# 改了 pyproject.toml 或 Dockerfile
./scripts/deploy.sh --rebuild-backend
```

### 3.6 环境变量说明

生产使用 `deploy/.env`，本地开发使用 `knowledge-service/.env`（`localhost` 连接）。

| 变量 | 必填 | 说明 |
|------|------|------|
| `LLM_API_KEY` | 是 | DashScope API Key |
| `JWT_SECRET_KEY` | 是 | JWT 签名密钥（≥16 字符） |
| `OSS_*` | 上传时需要 | 阿里云 OSS 配置 |
| `DATABASE_URL` 等 | 是 | 生产环境用 Docker 服务名（postgres、redis 等） |

完整变量见 `deploy/.env.example`。

## 四、本地开发

### 4.1 启动步骤

```bash
# 后端
cd knowledge-service
cp .env.example .env   # 填 LLM_API_KEY、JWT_SECRET_KEY
uv venv --python 3.12 .venv && uv pip install -e .
./scripts/start.sh     # 启动 Docker 依赖 + API + Worker

# 前端（新终端）
cd frontend
npm install && npm run dev
```

访问 http://localhost:5173 ，注册或登录即可。

### 4.2 导入测试数据（可选）

```bash
cd knowledge-service
./scripts/seed_platform_data.py
# 导入 docs/data 下的法规、类案、合规示例文档
```

## 五、常见问题

### Q1：访问 IP 返回 403

**原因**：`deploy/app/` 无前端文件（缺少 `index.html`）。

**解决**：执行 `./scripts/deploy.sh` 构建前端，或手动 `npm run build` 后复制到 `deploy/app/`。

### Q2：登录提示「邮箱或密码错误」

**原因**：生产库无用户，或密码不正确。

**解决**：运行 `./scripts/bootstrap_user.sh` 创建账号；密码至少 8 位。

### Q3：注册提示密码至少 8 位

**原因**：后端校验 `password` 长度 ≥ 8。

**解决**：使用 `seed12345` 等 8 位以上密码，不要用 `admin`。

### Q4：文档一直 processing

**原因**：Celery Worker 未运行或处理失败。

**解决**：

```bash
docker compose logs worker
docker compose restart worker
```

### Q5：Postgres unhealthy

**原因**：常见为数据目录版本不兼容（如 PG16 数据配 PG17 镜像）。

**解决**：

```bash
docker compose down
sudo rm -rf data/postgres/*
docker compose up -d
```

### Q6：上传文档失败

**原因**：未配置 OSS 或 Key 错误。

**解决**：在 `deploy/.env` 中配置 `OSS_ACCESS_KEY_ID`、`OSS_ACCESS_KEY_SECRET`、`OSS_ENDPOINT`、`OSS_BUCKET_NAME`。

### Q7：问答无引用、检索为空

**原因**：文档未处理完成，或助手未绑定知识库。

**解决**：确认文档 status=done；助手设置中绑定对应知识库。
