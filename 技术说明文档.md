# 枫桥智诉 · 技术说明文档

本文档说明系统**技术选型**、**整体架构**及**各模块实现**，供开发与运维人员参考。

---

## 一、技术选型总览

### 1.1 选型原则

| 原则 | 说明 |
|------|------|
| 开源可部署 | 核心组件均可私有化部署，不绑定单一云厂商 |
| OpenAI 兼容 | LLM / Embedding 通过 LiteLLM 对接 DashScope 等兼容接口，便于切换模型 |
| 多存储分工 | 关系型、向量、图、缓存、对象存储各司其职 |
| 异步解耦 | 文档入库等耗时任务走 Celery，API 快速响应 |

### 1.2 技术栈一览

| 层次 | 技术 | 版本/说明 | 选型理由 |
|------|------|-----------|----------|
| **前端** | Vue 3 | Composition API | 生态成熟，与 Element Plus 配合快 |
| | Element Plus | UI 组件库 | 企业后台常用，表单/表格完善 |
| | Pinia | 状态管理 | 轻量，替代 Vuex |
| | Vite | 构建工具 | 开发热更新快 |
| | D3.js | 图谱可视化 | 知识图谱力导向布局 |
| **后端** | FastAPI | Python 3.12 | 异步、OpenAPI 自动生成、类型友好 |
| | SQLAlchemy 2.0 | ORM | 异步 Session，Alembic 迁移 |
| | Celery | 任务队列 | 文档解析/向量化/图谱构建 |
| | LiteLLM | LLM 统一接口 | 一套代码对接多厂商 OpenAI 兼容 API |
| **关系库** | PostgreSQL + pgvector | PG 16 | 业务数据、用户、文档元数据；pgvector 可扩展向量能力 |
| **向量库** | Qdrant | v1.12 | 专用向量检索，Hybrid 检索 Dense 路 |
| **图数据库** | Neo4j | 5.21 Community | 实体关系存储，GraphRAG 遍历 |
| **缓存/队列** | Redis | 7 | Celery Broker、结果后端、会话辅助 |
| **对象存储** | 阿里云 OSS | — | 文档原文件存储，无本地磁盘兜底 |
| **反向代理** | Nginx | Alpine | 静态资源 + `/api`、`/ws` 反代 |
| **容器化** | Docker Compose | — | 一键部署，内网服务名互通 |

### 1.3 模型与 AI 能力

| 能力 | 默认模型/方案 | 接口 |
|------|---------------|------|
| 对话 / 报告 / 抽取 | DashScope `qwen-plus`（可配置） | OpenAI 兼容 `/v1/chat/completions` |
| 向量化 | `text-embedding-v3`，1024 维 | OpenAI 兼容 `/v1/embeddings` |
| 重排序 | `qwen3-rerank` | DashScope `/compatible-api/v1/reranks` |
| 文档解析 | unstructured | PDF/DOCX/MD 本地解析 |
| 稀疏检索 | rank-bm25 | 经典 BM25，非深度学习 |

## 二、系统架构

### 2.1 逻辑架构

```
┌─────────────────────────────────────────────────────────────────┐
│                         浏览器 (Vue SPA)                          │
└───────────────────────────────┬─────────────────────────────────┘
                                │ HTTP / WebSocket
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│  Nginx (:80)  ── 静态 app/  │  /api/*  /ws/*  ──►  FastAPI (:8000) │
└───────────────────────────────┬─────────────────────────────────┘
                                │
        ┌───────────────────────┼───────────────────────┐
        ▼                       ▼                       ▼
┌───────────────┐       ┌───────────────┐       ┌───────────────┐
│  PostgreSQL   │       │    Qdrant     │       │    Neo4j      │
│  业务/切块    │       │  向量检索     │       │  知识图谱     │
└───────────────┘       └───────────────┘       └───────────────┘
                                │
                                ▼
                        ┌───────────────┐
                        │ Celery Worker │
                        │ ingest / graph│
                        └───────┬───────┘
                                │
                        ┌───────▼───────┐
                        │     Redis     │
                        │  任务队列     │
                        └───────────────┘
                                │
                        ┌───────▼───────┐
                        │  阿里云 OSS   │
                        │  文档原文件   │
                        └───────────────┘
                                │
                        ┌───────▼───────┐
                        │  DashScope    │
                        │ LLM/Embed/Rerank│
                        └───────────────┘
```

### 2.2 部署架构（Docker Compose）

| 容器 | 镜像 | 职责 |
|------|------|------|
| `fengqiao-nginx` | nginx:alpine | 对外 :80，静态 + 反代 |
| `fengqiao-api` | fengqiao-api:latest | FastAPI，同步 HTTP/WS |
| `fengqiao-worker` | 同上 | Celery Worker，异步流水线 |
| `fengqiao-postgres` | pgvector/pg16 | 关系数据 |
| `fengqiao-qdrant` | qdrant:v1.12.4 | 向量库 |
| `fengqiao-neo4j` | neo4j:5.21-community | 图库 |
| `fengqiao-redis` | redis:7-alpine | 队列 |

`api` 与 `worker` **共用同一镜像**，通过不同 `command` 区分进程。

### 2.3 多租户与鉴权

- **JWT**：Access Token + Refresh Token，HS256 签名
- **租户隔离**：`tenant_id` 贯穿 PostgreSQL、Qdrant filter、Neo4j 子图
- **文档范围**：`scope=platform`（平台公共库）/ `scope=user`（用户私有）
- **角色**：`user` / `admin`，管理员可访问检索测试、图谱、系统配置

### 3.4 存储层 (`storage/`)

| 模块 | 技术 | 存储内容 |
|------|------|----------|
| `postgres.py` | async SQLAlchemy | 用户、租户、数据集、文档、切块、会话、消息、报告、助手 |
| `postgres_sync.py` | psycopg2 | Celery 同步路径（避免 asyncio 跨 loop 问题） |
| `qdrant_client.py` | qdrant-client | 子块向量，payload 含 tenant_id、dataset_id、doc_type |
| `neo4j_client.py` | neo4j driver | 实体、关系、社区、Local/Global Search |
| `oss.py` | oss2 | 文档原文件上传下载 |

### 3.5 入库流水线 (`pipeline/` + `ingest/`)

**流程**：

```
上传 → OSS 存原文件 → Celery ingest 任务
  → unstructured 解析 (parser.py)
  → 父子切块 (chunker.py, strategies.py)
  → Embedding 写入 Qdrant
  → 切块元数据写入 PostgreSQL
  → graph_build：LLM 抽取实体关系 → Neo4j
  → （可选）community 社区发现
```

**父子块 RAG**：

- **子块**：体积小（默认 300 tokens），用于向量/BM25 命中
- **父块**：上下文完整（默认 1200 tokens），命中后拼入 Prompt

### 3.6 检索模块 (`retrieve/`)

**Hybrid 检索链路**：

```
用户 Query
    │
    ├─► Dense：Qdrant 余弦相似（text-embedding-v3）
    │
    └─► Sparse：BM25（rank-bm25，租户内建索引）
              │
              ▼
         RRF 融合（Reciprocal Rank Fusion）
              │
              ▼
         qwen3-rerank 重排（Top rerank_top_k → search_top_k）
              │
              ▼
         附加父块上下文 → 返回 citations
```

| 文件 | 职责 |
|------|------|
| `hybrid.py` | 双路召回、RRF、调 Reranker、查父块、60s 结果缓存 |
| `bm25.py` | 租户级 BM25 索引构建与检索 |
| `reranker.py` | 调用 DashScope rerank API，失败降级为 RRF 顺序 |

**可调参数**（系统配置或 `.env`）：

| 参数 | 默认 | 作用 |
|------|------|------|
| `search_top_k` | 10 | 最终返回条数 |
| `dense_top_k_multiplier` | 3 | Dense/BM25 每路扩展倍数 |
| `rrf_k` | 60 | RRF 平滑系数 |
| `rerank_top_k` | 20 | 送入重排的候选数 |
| `rerank_model_id` / `rerank_instruct` | qwen3-rerank | 重排模型与指令 |
| `bm25_k1` / `bm25_b` | 1.5 / 0.75 | BM25 参数 |

### 3.7 Skills 模块 (`skills/`)

模块化业务能力，通过 `registry.py` 注册，供 API / MCP 调用。

| Skill | 目录 | 功能 |
|-------|------|------|
| 文档处理 | `document/` | 文档相关操作 |
| RAG 检索 | `rag_search/` | 封装混合检索 |
| 法规检索 | `search_law/` | 法规库定向检索 |
| 类案检索 | `search_case/` | 类案库定向检索 |
| 用户文档 | `search_user_docs/` | 用户材料检索 |
| 问答 | `qa/` | 检索增强问答 |
| 报告生成 | `report_generation/` | 结构化报告 + 规则引擎 |
| 规则检查 | `check_rules/` | YAML 规则（如劳动合规） |
| 图谱查询 | `graph_query/` | Neo4j 局部/全局检索 |
| 用户报告 | `get_user_report/` | 读取用户报告上下文 |

### 3.8 WebSocket 问答 (`api/routes/ws.py`)

1. 客户端连接 `/ws/chat?token=...`
2. 从助手读取绑定的 `dataset_ids`、`report_ids`
3. `retrieve_children` 混合检索
4. 拼装 Prompt，`stream_chat` 流式输出
5. 生成推荐追问，写入消息与 citations

温度等参数从 `settings` 读取（可系统配置在线调整）。

## 五、数据模型（概要）

| 实体 | 主要字段 | 说明 |
|------|----------|------|
| Tenant | name | 租户 |
| User | email, role, tenant_id | 用户 |
| Dataset | name, metadata.scope | 知识库 |

## 七、扩展与集成

### 7.1 MCP Server

`adapters/mcp_server.py` 提供 MCP 协议接入，便于外部 Agent 调用检索、问答等能力。

### 7.2 数据库迁移

```bash
cd knowledge-service
.venv/bin/alembic upgrade head
```

### 7.3 镜像构建缓存

Dockerfile 分层：

1. 系统依赖（apt）
2. 仅 `pyproject.toml` → `pip install` 依赖（**缓存层**）
3. `COPY src` → `pip install --no-deps -e .`（源码变更只重建此层）
